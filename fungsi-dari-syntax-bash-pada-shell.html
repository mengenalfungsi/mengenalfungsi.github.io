16. Buatlah perintah shell (bash) untuk membuat 200 file kosong dengan format penamaan YYYYMMDD-AAA.txt pada direktori kerja, di mana YYYYMMDD adalah tanggal pada saat perintah/script dieksekusi, dan AAA adalah 3 digit nomor urut, dari 001 hingga 200. Jelaskan perintah/script tersebut, dan lampirkan hasilnya.


Perintah shell (bash) untuk persoalan ini adalah:

Multiline:
tgl=$(date +%Y%m%d)
seq -w 1 200 | while read -r nomor; do
   namafile="$tgl-$nomor.txt"
   echo "Membuat file $namafile..."
   touch ./$namafile
doneSingle-line (one-liner):
tgl=$(date +%Y%m%d); seq -w 1 200 | while read -r nomor; do namafile="$tgl-$nomor.txt"; echo "Membuat file $namafile..."; touch $namafile; done

 

Pembahasan

Sesuai deskripsi persoalan, kita membutuhkan string-string berikut ini.

String tanggal hari ini dalam format YYYYMMDD, yang dapat diperoleh dari hasil eksekusi perintah shell: date +%Y%m%d.Kumpulan string nomor urut file dari "001" hingga "200", yang dapat diperoleh dari hasil eksekusi perintah shell: seq -w 1 200.
Parameter -w digunakan agar setiap string yang dihasilkan memiliki panjang yang sama dengan angka batas terbesar, dengan penyisipan "0" di depan string jika diperlukan.

Pada dasarnya, perintah shell yang diperlukan adalah:
tgl=$(date +%Y%m%d)
seq -w 1 200 | while read -r nomor; do
   touch ./$tgl-$nomor.txt
done

Untuk mode one-liner:
tgl=$(date +%Y%m%d); seq -w 1 200 | while read -r nomor; do touch ./$tgl-$nomor.txt; done

Perintah tersebut melakukan:

Inisialisai variabel tgl dengan tanggal hari ini berformat YYYYMMDD.
Hal ini dilakukan sebelum masuk ke dalam iterasi while, bukan di dalam while, agar perintah date hanya dipanggil sekali saja.Pipeline (mengarahkan nilai) kumpulan string nomor urut (yang dihasilkan dari perintah seq) ke perintah while read -r nomor, sehingga untuk setiap iterasi, variabel nomor berisi nomor urut.Setiap iterasi melakukan pembuatan file kosong dengan format YYYYMMDD-AAA.txt dengan perintah touch.

Untuk membuat sebuah file kosong, digunakan touch, bukan echo > namafile, karena dengan perintah echo > namafile, file yang dibuat akan berisi 1 karakter null, yang mengakibatkan ukurannya menjadi 1 Byte. Jadi, file yang dibuat tidak kosong, dan tidak sesuai dengan apa yang diminta.

Sebagai pelengkap, tangkapan layar dari daftar isi direktori kerja dapat dilihat pada gambar.

Penjelasan tentang syntax

______________

Syntax merupakan salah satu ilmu kebahasaan yang mempelajari tentang struktur pembentukan kata-kata secara gramatikal sehingga menjadi satu kesatuan kalimat terstruktur dan bermakna jelas.

Contoh :

I eat an apple  

Kata "I" merupakan subyek yang harus diletakkan di awal kalimat.

Kata "eat" merupakan kata kerja yang harus diletakkan setelah subyek

determiner "an" menyatakan jumlah ( kuantitas ). Ini bagian dari modifikasi kata benda (obyek ) yang dijabarkan.

kata "apple" merupakan obyek yang harus diletakkan setelah kata kerja.

Contoh kalimat diatas sudah sesuai dengan penyusunan kata-kata secara gramatikal. Jika disusun dalam pohon diagram syntax akan terlihat runtut. Sebuah kalimat lengkap secara sintaksis terdiri dari NP (  Noun Phrase/FRASA KATA BENDA) + VP ( Verb Phrase/FRASA KATA KERJA ).

NP => sekumpulan kata -kata yang memiliki satu makna yang berfungsi memodifikasi kata bendanya .

VP => sekumpulan kata yang terdiri dari kata kerja dan objek

24. fungsi syntax union digunakan untuk


Fungsi Syntax Union digunakan untuk melakukan penggabungan query dalam melakukan pemanggilan select. Syntax Union sendiri berkaitan dengan penggunaan dalam MySQL. Seperti halnya dalam matematika, syntax union akan menghilangkan duplikasi hasil dari pemanggilan select. Dalam pemanggilan, unsur dalam expression yang boleh berubah hanyalah nama tabel.

Pembahasan

Dalam penggunaan fungsi syntax union, ada beberapa hal yang harus diperhatikan antara lain:

Nama tabel dapat berbeda antar SELECT, namun, harus ada minimal satu nama tabel di salah satu expression SELECT.Jumlah field antara SELECT awal dan lainnya harus sama. Contoh: Pada select pertama terdapat 3 field, maka select lainnya harus juga 3 field.Syntax Union secara otomatis akan menghapus setiap duplikasi yang muncul pada hasil sehingga tidak diperlukan suatu pengubah distinctJika menggunakan pernyataan ORDER, maka field pada setiap SELECT yang diisikan berupa alias / nama lain bukan nama asliPenggunaan pernyataan ORDER yang tidak diikuti dengan pernyataan LIMIT harus diletakkan pada susunan terakhir agar dapat terbaca

26. 2. Apakah Syntax yang digunakan dalam fungsi AVERAGEIF! Jawab: ...​


Syntax yang digunakan dalam fungsi AVERAGEIF adalah sebagai berikut:

AVERAGEIF(range, criteria, [average_range])

- range: Rentang sel yang akan dievaluasi berdasarkan kriteria tertentu.

- criteria: Kriteria yang digunakan untuk menentukan data mana yang akan dihitung rata-ratanya.

- average_range (opsional): Rentang sel yang berisi data yang ingin dihitung rata-ratanya. Jika tidak ditentukan, maka range akan digunakan sebagai average_range.

Contoh penggunaan fungsi AVERAGEIF:

Misalnya, kita memiliki rentang sel A1:A5 yang berisi nilai-nilai, dan kita ingin menghitung rata-rata hanya dari nilai-nilai yang lebih besar dari 5.

AVERAGEIF(A1:A5, ">5")

Dalam contoh di atas, range adalah A1:A5, dan kriteria adalah ">5". Fungsi ini akan menghitung rata-rata dari nilai-nilai yang memenuhi kriteria tersebut.

Catatan: Syntax dan cara penggunaan fungsi AVERAGEIF dapat bervariasi tergantung pada program spreadsheet atau bahasa pemrograman yang digunakan. Contoh di atas mengacu pada penggunaan fungsi AVERAGEIF dalam aplikasi spreadsheet seperti Microsoft Excel atau Google Sheets.

Perintah untuk membuat user dengan nama tool dengan direktori /home/toor123 dengan shell bash, UID 110, GID 110, dilengkapi dengan komentar "user ini baru lho"​ adalah:

Pembahasan

Asumsi awal: Sistem operasi yang digunakan adalah Linux.

Pada sistem operasi Linux, untuk menambahkan user, terdapat 2 buah perintah yang dapat digunakan, yaitu [tex]\tt adduser[/tex] dan [tex]\tt useradd[/tex].
Pada beberapa distro Linux, [tex]\tt adduser[/tex] adalah symbolic link ke [tex]\tt useradd[/tex]. Namun, tidak demikian pada beberapa distro Linux lainnya.

adduser merupakan script Perl untuk menambahkan user secara “lebih aman”, karena tanpa parameter apapun selain username, [tex]\tt adduser[/tex] akan membuat home directory untuk user tersebut sesuai username, menyalin file-file default untuk user yang dispesifikasikan pada [tex]\tt /etc/skel[/tex], dan menampilkan wizard untuk melengkapi informasi user.useradd merupakan perintah low-level untuk menambahkan user. Tanpa parameter flag/opsi apapun, [tex]\tt useradd[/tex] hanya akan menambahkan user saja, tanpa melakukan hal lain yang dilakukan secara default oleh perintah [tex]\tt adduser[/tex].

Detail tentang sintaks dan cara penggunaan kedua perintah dapat dibaca pada help atau manualnya.

PENYELESAIAN

Yang akan kita lakukan adalah membuat user dengan rincian:

nama (username) = tooldirektori (home directory) = /home/toor123shell (login shell) = bash
Untuk shell, kita harus memberikan lokasi absolut [tex]\tt bash[/tex] yang terdapat sistem. Pada umumnya, file binary [tex]\tt bash[/tex] terdapat di /bin/bash, atau /sbin/bash. Tidak menutup kemungkinan terdapat di lokasi lain pada Linux yang sudah customized.
Lokasi absolut ini dapat ditentukan dengan perintah [tex]\texttt{which bash}[/tex].
Oleh karena itu, selain menggunakan lokasi absolut untuk login shell, agar tidak salah, kita dapat menggunakan [tex]\texttt{\$(which bash)}[/tex].UID (user ID) = 110GID (group ID) = 110komentar = "user ini baru lho"​

Cara 1: Dengan perintah “adduser”

Parameter yang digunakan:

Home directory
[tex]\texttt{--home /home/toor123}[/tex]Shell
[tex]\texttt{--shell \$(which bash)}[/tex]User ID
[tex]\texttt{--uid 110}[/tex]Group ID
[tex]\texttt{--gid 110}[/tex]Komentar
[tex]\texttt{--gecos "user ini baru lho"}[/tex]Username: [tex]\tt tool[/tex]

Asumsi: group dengan GID=110 belum ada.

Dengan adduser, kita harus membuat groupnya dengan [tex]\texttt{adduser --group --gid 110 tool}[/tex] atau [tex]\texttt{addgroup --gid 110 tool}[/tex]. Pakai saja nama group yang sama dengan nama user.

Kemudian, karena adduser akan menanyakan password untuk user yang dibuat, jika ingin user ini tanpa password, kita tambahkan parameter [tex]\texttt{--disabled-password}[/tex].

Jadi, perintah lengkapnya adalah:
addgroup --gid 110 tool && adduser --home /home/toor123 --shell $(which bash) --uid 110 --gid 110 --gecos "user ini baru lho" --disabled-password tool

Perintah tersebut harus dijalankan sebagai root. Jika bukan root, tambahkan sudo di depan perintah.

Outputnya kira-kira seperti ini:

[tex]\begin{aligned}&\texttt{Adding group 'tool' (GID 110) ...}\\&\tt Done.\\&\texttt{Adding user 'tool' ...}\\&\texttt{Adding new user 'tool' (110) with group 'tool' ...}\\&\texttt{Creating home directory '/home/toor123' ...}\\&\texttt{Copying files from '/etc/skel' ...}\end{aligned}[/tex]

______________
Cara 2: Dengan perintah “useradd”

Parameter yang digunakan:

Home directory
[tex]\texttt{--home-dir /home/toor123}[/tex] atau [tex]\texttt{-d /home/toor123}[/tex]
Perlu menambahkan parameter [tex]\texttt{--create-home}[/tex] atau [tex]\texttt{-m}[/tex] agar home directory dibuat jika belum ada.Shell
[tex]\texttt{--shell \$(which bash)}[/tex] atau [tex]\texttt{-s \$(which bash)}[/tex]User ID
[tex]\texttt{--uid 110}[/tex] atau [tex]\texttt{-u 110}[/tex]Group ID
Jika GID 110 sudah ada, kita gunakan parameter [tex]\texttt{--gid 110}[/tex] atau [tex]\texttt{-g 110}[/tex].
Namun, dengan asumsi GID 110 belum ada, kita gunakan parameter [tex]\texttt{--user-group}[/tex] atau [tex]\texttt{-U}[/tex]. Dengan parameter ini, useradd akan membuat group baru dengan GID=UID dan nama group sama dengan nama user.Komentar
[tex]\texttt{--comment "user ini baru lho"}[/tex] atau  [tex]\texttt{-c "user ini baru lho"}[/tex]Username: [tex]\tt tool[/tex]

Perintah lengkapnya adalah:
useradd --create-home --home-dir /home/toor123 --shell $(which bash) --uid 110 --user-group --comment "user ini baru lho" tool

Versi pendeknya:
useradd -m -d /home/toor123 -s $(which bash) -u 110 -U --comment "user ini baru lho" tool

Perintah tersebut harus dijalankan sebagai root. Jika bukan root, kita harus menambahkan sudo di depan perintah.

Jika sukses, tidak ada output yang ditampilkan.
