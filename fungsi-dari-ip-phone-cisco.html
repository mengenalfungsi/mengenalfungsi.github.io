8. TUGAS PROYEK Kerjakan tugas kelompok berikut dengan baik dan benar! Perhatikan gambar konsep berikut! Call Setup (Skinny) IP Phone Cisco CallManager Call Setup (H.323) H.323 Gateway Voice Traffic PSTN Phone Berdasarkan gambar konsep di atas, buatlah langkah-langkah kerja Softswitch yang benar! Sumber-sumber data bisa diambil dari media cetak dan elektronik yang relevan. Hasilnya dikumpulkan pada guru untuk mendapat penilaian!​


Jawaban:

Berikut adalah langkah-langkah kerja Softswitch yang benar berdasarkan gambar konsep yang diberikan:

1. Ketika IP Phone Cisco ingin melakukan panggilan, sinyal panggilan (call setup signal) akan dikirim melalui protokol Skinny ke Softswitch.

2. Softswitch akan meneruskan sinyal panggilan ke CallManager untuk diproses.

3. CallManager akan mencari informasi terkait nomor telepon yang dihubungi dan lokasi perangkat tujuan.

4. CallManager akan memberikan informasi tersebut ke Softswitch.

5. Softswitch akan meneruskan sinyal panggilan ke H.323 Gateway.

6. H.323 Gateway akan meneruskan sinyal panggilan ke PSTN (Public Switched Telephone Network) untuk melakukan panggilan ke nomor tujuan.

7. Jika panggilan berhasil dilakukan, suara dari nomor telepon tujuan akan dikirim ke PSTN.

8. PSTN akan mengirimkan suara tersebut ke H.323 Gateway.

9. H.323 Gateway akan mengirimkan suara tersebut ke Softswitch.

10. Softswitch akan meneruskan suara tersebut ke IP Phone Cisco yang melakukan panggilan.

11. Panggilan berhasil dilakukan dan kedua pihak dapat berkomunikasi.

Dengan langkah-langkah kerja Softswitch yang benar di atas, sistem panggilan suara dapat berjalan dengan baik dan benar.

17. Apa Yang Di Konfigurasi Di IP Phone Agar Bisa Melakukan Panggilan??Pliss Jawaban kak Bsok di Kumpul kan​


Berikut adalah langkah-langkah umum yang diperlukan:

Pengaturan Jaringan: Pertama, Anda perlu mengkonfigurasi pengaturan jaringan pada telepon IP Anda. Ini biasanya mencakup pengaturan alamat IP (mungkin menggunakan DHCP), gateway, subnet mask, dan DNS server. Jika perangkat Anda berada di belakang firewall atau NAT, Anda mungkin perlu mengkonfigurasi pengaturan tambahan.

Pendaftaran ke VoIP Provider atau PBX: Anda perlu mendaftarkan telepon IP Anda ke layanan VoIP atau sistem PBX (Private Branch Exchange) yang Anda gunakan. Ini biasanya melibatkan pengaturan seperti alamat server, nama pengguna (biasanya dalam bentuk nomor telepon atau extensi), dan kata sandi. Informasi ini biasanya disediakan oleh penyedia layanan VoIP atau administrator PBX Anda.

Pengaturan Codec: Codec adalah algoritma yang digunakan untuk mengubah suara menjadi data digital dan sebaliknya. Anda mungkin perlu mengkonfigurasi codec yang diizinkan pada telepon IP Anda. Biasanya, codec seperti G.711, G.722, atau G.729 digunakan.

Pengaturan Dial Plan: Dial Plan adalah aturan yang digunakan oleh telepon IP Anda untuk menentukan bagaimana menerjemahkan nomor yang Anda masukkan ke dalam alamat yang dapat dipanggil. Misalnya, Anda mungkin perlu menambahkan awalan tertentu untuk panggilan internasional.

Pengaturan QoS (Quality of Service): Untuk memastikan kualitas suara yang baik, Anda mungkin perlu mengatur QoS pada perangkat jaringan Anda (seperti router atau switch) untuk memberikan prioritas trafik suara dibandingkan trafik lain.

Pengaturan lainnya: Ada juga berbagai pengaturan lain yang mungkin perlu Anda lakukan, tergantung pada perangkat dan layanan yang Anda gunakan. Ini bisa termasuk pengaturan seperti konferensi panggilan, panggilan menunggu, penerusan panggilan, dan lain-lain.

27. apa itu mikrotik dan cisco


Mikrotik adalah sistem operasi dan perangkat lunak yang dapat digunakan untuk menjadikan komputer manjadi router network yang handal, mencakup berbagai fitur yang dibuat untuk IP network dan jaringan wireless, cocok digunakan oleh ISP, provider hotspot dan warnet.

sedangkan  Cisco adalah peralatan utama yang banyak digunakan pada Jaringan Area Luas atau Wide Area Network (WAN). Dengan cisco router, informasi dapat diteruskan ke alamat-alamat yang berjauhan dan berada di jaringan computer yang berlainan. Yang bertujuan untuk dapat meneruskan paket data dari suatu LAN ke LAN lainnya, Cisco router menggunakan tabel dan protocol routing yang berfungsi untuk mengatur lalu lintas data. Paket data yang tiba di router diperiksa dan diteruskan ke alamat yang dituju. Agar paket data yang diterima dapat sampai ke tujuannya dengan cepat, router harus memproses data tersebut dengan sangat tepat. Untuk itu, Cisco Router menggunakan Central Processing Unit (CPU) seperti yang digunakan di dalam komputer untuk memproses lalu lintas data tersebut dengan cepat. Seperti komputer, cisco router juga mempunyai sejumlah jenis memori yaitu ROM, RAM, NVRAM dan FLASH, yang berguna untuk membantu kerjanya CPU. Selain itu dilengkapi pula dengan sejumlah interface untuk berhubungan dengan dunia luar dan keluar masuk data. Sistem operasi yang digunakan oleh cisco router adalah Internetwork Operating System (IOS).

40. Gan tolong bantu dong cara setting ip router static untuk skema seperti di gambar dengan metode CLI di Cisco Packet Tracer. Terimakasih


Jawaban:

Tentu, saya akan memberikan langkah-langkah untuk mengatur IP router secara statis menggunakan metode CLI di Cisco Packet Tracer berdasarkan skema yang diberikan dalam gambar. Berikut adalah langkah-langkahnya:

1. Buka Cisco Packet Tracer dan buat skenario sesuai dengan gambar yang diberikan. Pastikan Anda memiliki router yang sesuai dengan model yang digunakan dalam gambar.

2. Klik dua kali pada router untuk membuka jendela konfigurasi CLI.

3. Masukkan mode konfigurasi dengan mengetikkan perintah berikut:

```

enable

configure terminal

```

4. Atur alamat IP pada antarmuka FastEthernet 0/0 dengan mengetikkan perintah berikut:

```

interface FastEthernet 0/0

ip address 192.168.1.1 255.255.255.0

```

5. Aktifkan antarmuka FastEthernet 0/0 dengan perintah berikut:

```

no shutdown

```

6. Atur alamat IP pada antarmuka Serial 0/0/0 dengan mengetikkan perintah berikut:

```

interface Serial 0/0/0

ip address 10.0.0.1 255.255.255.0

```

7. Aktifkan antarmuka Serial 0/0/0 dengan perintah berikut:

```

no shutdown

```

8. Keluar dari konfigurasi antarmuka dengan mengetikkan perintah berikut:

```

exit

```

9. Simpan konfigurasi dengan mengetikkan perintah berikut:

```

copy running-config startup-config

```

10. Konfigurasi IP statis pada router selesai. Anda dapat mengecek konfigurasi dengan menggunakan perintah `show running-config` atau `show ip interface brief`.

Pastikan Anda mengganti alamat IP dengan yang sesuai untuk jaringan Anda. Selain itu, pastikan kabel sudah tersambung dengan benar antara router dan perangkat lain sesuai dengan skema yang diberikan.

Harap dicatat bahwa langkah-langkah ini berdasarkan penggunaan Cisco Packet Tracer, dan konfigurasi dapat sedikit berbeda jika menggunakan perangkat nyata atau simulator lainnya.
