1. jelaskan sejarah aplikasi samba


 



Sejarah Lahirnya Samba


Bertahun lalu, di masa awal PC, IBM dan Sytec mengembangkan sebuah sistem network yang disebut NetBIOS (Network Basic Input Operating System). NetBIOS merupakan software yang menyediakan interface antara program dan hardware network. Pengalamatan dalam NetBIOS menggunakan nama 16 bit yang sekarang dikenal sebagai NetBIOS Name. Di dalam lingkungan Windows NT, NetBIOS Name disebut juga Computer Name. 

Kemudian Microsoft menambahkan sebuah feature berupaI/O redirection, sehingga sebuah resource local (printer, hard disk) bisa diakses melalui network oleh komputer lain, dengan bentuk, bau dan rasa seperti recource local di komputer yang sedang mengakses tersebut. Jika anda pengguna LAN Manager atau MS-DOS Client, anda tentu familiar dengan command "Net use drive: \\computer name\share name" atau pengguna Windows dengan perintah Map Network Drive. Keduanya membuat saturemote resource di komputer yang menjalankan perintah tersebut seolah-olah berada di komputer itu.

Microsoft menyebut penambahan feature ini sebagai NetBEUI. Hal ini yang kemudian disalah artikan bahwa NetBEUI merupakan protokol standar dalam lingkungan network dari Microsoft. Padahal nama dari I/O Redirectiondari NetBEUI adalah SMB (Server Message Block) atau menurut istilah Microsoft CIFS (Common Internet File System). Keterangan lengkap mengenai protokol SMB ini bisa dibaca di RFC 1001 dan 1002.


Dalam protokol standar LAN Manager dan Windows (NetBEUI, alias SMB, alias CIFS), hal terpenting yang menjadi urat nadinya adalah kemampuan file dan print sharing, serta kemampuan browsing. Kemudian dengan munculnya NT, ditambahkan dua feature tambahan yaituauthentication dan authorization untuk setiap servis yang akan diakses. Jadi jika kita bicara NetBEUI, sebenarnya ada 2 komponen yang terdapat di dalamnya; (1) NetBIOS sebagai pengenal komputer yang satu dengan lainnya dan (2) I/O Redirection sebagai fasilitas untuk memberikan share dan mengakses shared resources komputer lain.


      Bulan Desember 1991, merupakan sejarah kelahiran Samba Server, yang dibuat oleh seorang Andrew Tridgell, mahasiswa PhD (Doktor) Ilmu Komputer di Australian National University (ANU), Canberra, Australi. Singkatya, tahun 1992 project Samba dibuat untuk mengatasi persoalan yang muncul dikala dia hendak menghubungkan komputer miliknya (Linux) dengan komputer (Ms.Windows). (Sebenarnya samba telah dibuat sebelumnya namun belum diporting ke Sistem Operasi Linux.  Tahun 1992, merupakan awalnya berdiri team Samba yang pada waktu itu beranggotakan Jeremy Allison, Jochen Hupert, Matthew Harrell, Frank Varnavas dan beberapa hacker lain yang tersebar diseluruh dunia.

8. makna dibuatnya tari samba adalah?


Samba (lafal: [sɐbɐ]) adalah tarian Brasil dan genre musik yang berakar dari Afrika. Hal ini diakui di seluruh dunia sebagai simbol dari Brasil dan Karnaval Brasil. Dianggap sebagai salah satu ungkapan paling populer budaya Brasil, samba telah menjadi ikon identitas nasional Brasil. Samba de Roda (tari lingkaran) dari Bahia, yang menjadi warisan dunia oleh UNESCO bidang kemanusiaan pada tahun 2005, adalah akar utama dari Carioca samba, samba yang dimainkan dan ditarikan di Rio de Janeiro.

Secara tradisional, samba dimainkan dengan senar (cavaquinho dan berbagai jenis gitar) dan berbagai instrumen perkusi seperti tamborim. Dengan pengaruh orkestra Amerika sejak Perang Dunia Kedua dan dampak budaya musik AS pascaperang, mulai digunakan juga instrumen tiup seperti trombon, terompet, choro, flute dan klarinet.

Samba adalah program populer yang gratis yang memungkinkan user untuk mengakses dan menggunakan file, printer, dan sumber berbagi lainnya dalam sebuah intranet ataupun internet. Samba sering dikaitkan sebagai sebuah sistem file jaringan dan dapat dipasang pada berbagai macam platform sistem operasi, termasuk Linux, platform UNIX, OpenVMS, dan OS/2.

Samba berbasis protokol client/server dari SMB (Server Message Block) dan CIFS (Common Internet File System). Ketika menggunakan software client yang juga mendukung SMB/CIFS (sebagai contoh, kebanyakan produk Windows), user mengirimkan sekumpulan client request ke server Samba pada komputer lain untuk membuka file, mengakses shared printer, atau mengakses resource yang lain. Server Samba pada komputer lain akan merespon setiap client request, apakah memberikan ataupun menolak akses ke shared files dan resource. Client pada SMB/CIFS Samba disebut sebagai smbclient.

Beberapa alasan yang membuat administrator sistem mengimplementasikan aplikasi samba adalah sebagai berikut

- Administrator tidak mau atau tidak mampu untuk membeli server Windows NT tetapi masih membutuhkan fungsionalitas dari server tersebut

- Administrasi ingin menyediakan area umum bagi data atau direktori user untuk transisi dari server Windows ke Unix, ataupun sebaliknya

- Administrator ingin membagi printer pada workstation Windows dan Unix

- Administrator ingin mengakses file NT dari server Unix.

12. Bagaimana cara mengintegrasikan Samba Server dengan Active Directory? Jelaskan langkah - langkahnya secara terperinci, termasuk konfigurasi Kerberos, LDAP, dan DNS, serta cara memastikan  bahwa koneksi antara Samba Server dan Active Directory berfungsi dengan  baik.tolong kak​


Jawaban:

1. Instalasi Samba Server:

- Unduh dan instal Samba Server terbaru.

- Pastikan bahwa Samba Server telah berhasil diinstal.

2. Konfigurasi Kerberos:

- Buat file konfigurasi Kerberos dengan nama krb5.conf.

- Tambahkan informasi tentang domain Active Directory dan nama server Kerberos.

- Simpan file konfigurasi Kerberos.

3. Konfigurasi LDAP:

- Buat file konfigurasi LDAP dengan nama ldap.conf.

- Tambahkan informasi tentang domain Active Directory dan nama server LDAP.

- Simpan file konfigurasi LDAP.

4. Konfigurasi DNS:

- Buat file konfigurasi DNS dengan nama named.conf.

- Tambahkan informasi tentang domain Active Directory dan nama server DNS.

- Simpan file konfigurasi DNS.

5. Konfigurasi Samba Server:

- Buat file konfigurasi Samba Server dengan nama smb.conf.

- Tambahkan informasi tentang domain Active Directory dan nama server Samba.

- Simpan file konfigurasi Samba Server.

6. Tes Koneksi:

- Jalankan perintah “smbclient” untuk memeriksa koneksi antara Samba Server dan Active Directory.

- Jika koneksi berhasil, maka Anda akan melihat pesan “Connected to domain”.

- Jika koneksi gagal, maka Anda akan melihat pesan “Connection failed”.

7. Verifikasi Koneksi:

- Jalankan perintah “net ads testjoin” untuk memverifikasi koneksi antara Samba Server dan Active Directory.

- Jika koneksi berhasil, maka Anda akan melihat pesan “Join is OK”.

- Jika koneksi gagal, maka Anda akan melihat pesan “Join failed”.
